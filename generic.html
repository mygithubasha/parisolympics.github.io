SQL EXPLANATION

In SQL, a subquery, also known as an inner query or nested query, is a query embedded within another query.
The subquery can be part of a `SELECT`, `INSERT`, `UPDATE`, or `DELETE` statement, or part of another subquery.
Its primary role is to provide a set of data that can be used in the main query.

Types of Subqueries

1. Single-row Subquery Returns a single row with one or more columns.
2. Multiple-row Subquery: Returns multiple rows with one or more columns.
3. Multiple-column Subquery: Returns multiple columns from a single row.
### Uses of Subqueries - **Filtering Results:** Used in the `WHERE` or `HAVING` clause to filter data based on the results of the subquery. - **Data Comparison:** Compare values with another set of values returned by the subquery. - **Inserting or Updating Data:** Insert data into a table or update data in a table based on the results of a subquery. ### Examples #### Example 1: Simple Subquery in `WHERE` Clause Find employees whose salary is above the average salary in the company: ```sql SELECT first_name, last_name FROM employees WHERE salary > (SELECT AVG(salary) FROM employees); ``` In this example, the subquery `(SELECT AVG(salary) FROM employees)` calculates the average salary. The main query then selects employees whose salary is greater than this average. #### Example 2: Subquery in `FROM` Clause (Derived Table) Find the highest salary in each department and list them along with department names: ```sql SELECT d.department_name, t.max_salary FROM departments d JOIN ( SELECT department_id, MAX(salary) AS max_salary FROM employees GROUP BY department_id ) t ON d.department_id = t.department_id; ``` Here, the subquery is used as a derived table to find the maximum salary per department, and the main query joins this result with the departments table to get the department names. ### Considerations - **Performance:** Subqueries can sometimes be slower than equivalent joins, especially if not properly indexed. - **Correlated vs. Non-Correlated:** A correlated subquery depends on the outer query and is evaluated once for each row processed by the outer query. A non-correlated subquery is independent and is evaluated only once. Understanding and using subqueries effectively can enhance your SQL queries' functionality and efficiency, especially for complex data retrieval tasks.
